Monitoring System

Ensuring system reliability through automated health checks and log analysis.

Simulation Logic

The monitoring script (`scripts/monitor.sh`) performs a 3-step validation cycle:

  • Container Health: Verifies the Docker engine and target container status.
  • Log Extraction: Retrieves the latest operational logs from the container.
  • Anomaly Detection: Scans for error patterns and resource spikes.

Alerting Mechanism

In the event of a failure, the system generates a simulated SNS alert and provides detailed diagnostic output to the pipeline orchestrator.
